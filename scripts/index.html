PC Setup Scripts

PowerShell scripts for the things that have to be re-fixed every time you sign into a new or freshly imaged machine. Each script picks the right mechanism at run time, so the same file works on both Windows versions.

How to run any of these

  • One-time use — nothing is left on the PC. Each script changes a Windows setting and exits, exactly like flipping the toggle by hand. Run it once, then delete the .ps1; only the setting persists. Nothing keeps running in the background. (The single exception is the tray script's optional -Install — skip it for one-and-done; see that note below.)
  • Save the .ps1 somewhere local, open PowerShell, then run: powershell -ExecutionPolicy Bypass -File .\NAME.ps1
  • -List shows what would change without touching anything · -WhatIf is a full dry run · -Enable reverses a disable.
  • Copied from a network share and blocked? Run Unblock-File .\NAME.ps1 once.

Step by step (example: Show-AllTrayIcons.ps1)

  1. Save the script somewhere simple. The root of the C: drive (C:\) is easiest because the path has no spaces. Click Download .ps1 on any script below, then move it to C:\ (or just use your Desktop).
  2. Open PowerShell. Press Start, type PowerShell, press Enter. No admin is needed for the tray or Copilot scripts; for the Malwarebytes service option, right-click Windows PowerShell and choose Run as administrator.
  3. Paste the line that matches where you saved it, then press Enter.

If you saved it to the root of C::

powershell -ExecutionPolicy Bypass -File C:\Show-AllTrayIcons.ps1

If you saved it to your Desktop:

powershell -ExecutionPolicy Bypass -File "$HOME\Desktop\Show-AllTrayIcons.ps1"

Swap Show-AllTrayIcons.ps1 for any other script name, and add switches on the end (e.g. ... .ps1 -List). Because the full path is in the command, you don't need to change folders first. OneDrive tip: if the Desktop line errors, your Desktop is synced to OneDrive — use "$HOME\OneDrive\Desktop\Show-AllTrayIcons.ps1" instead.

Taskbar

Show all system tray icons

Show-AllTrayIcons.ps1
No admin needed Download .ps1

Force every notification-area icon to show directly on the taskbar instead of hiding in the Windows 11 chevron flyout. Windows 11 made this per-app (IsPromoted under NotifyIconSettings) and new apps keep landing hidden; on Windows 10 it flips the old “always show all icons” switch. Once nothing is hidden the chevron disappears.

Show all icons now

Promotes every current tray icon and restarts Explorer so they appear immediately.

powershell -ExecutionPolicy Bypass -File C:\Show-AllTrayIcons.ps1
Do it manually instead

Run box: Win+R → ms-settings:taskbar opens Taskbar settings (Windows 11).

Windows 11
  1. Right-click the taskbar → Taskbar settings.
  2. Expand Other system tray icons.
  3. Turn On the toggle for each app you want always visible. The chevron disappears once nothing is hidden.

Faster: click the chevron to open the flyout, then drag an icon out onto the taskbar — that promotes it instantly.

Windows 10

Run box: Win+R → control /name Microsoft.NotificationAreaIcons opens the classic dialog with the “Always show all icons” checkbox.

  1. Right-click the taskbar → Taskbar settings.
  2. Under Notification area, click Select which icons appear on the taskbar.
  3. Turn on Always show all icons in the notification area.

Optional: re-apply automatically on every logon

This is the only option that leaves anything on the PC — skip it if you just want a one-time change. It copies the script to %LOCALAPPDATA% and registers a logon task that re-runs ~1 minute after each sign-in, so apps installed later get promoted too. Only worth it on a machine you manage long-term; for a one-and-done pass on a client PC, use the one-shot above instead.

powershell -ExecutionPolicy Bypass -File C:\Show-AllTrayIcons.ps1 -Install

Reverse: -UninstallRemoves the logon task and the local copy of the script.

Do it manually instead
  1. Open Task Scheduler → Create Task.
  2. Triggers tab: New → At log on, delay 1 minute.
  3. Actions tab: New → Program powershell.exe, arguments -NoProfile -ExecutionPolicy Bypass -WindowStyle Hidden -File "C:\path\Show-AllTrayIcons.ps1".
  4. Save. (There is no built-in Settings toggle that survives new app installs — this is the manual equivalent of -Install.)

Remove the logon task

Unregisters the task and deletes the local copy.

powershell -ExecutionPolicy Bypass -File C:\Show-AllTrayIcons.ps1 -Uninstall
Do it manually instead
  1. Open Task Scheduler, find ShowAllTrayIcons in the library, right-click → Delete.
  2. Delete the folder %LOCALAPPDATA%\ShowAllTrayIcons.
Startup

Stop Microsoft 365 Copilot at startup

Disable-Copilot-Startup.ps1
No admin needed Download .ps1

“Microsoft 365 Copilot” is the rebranded Microsoft 365 / Office app; “Copilot” is the standalone assistant. Both auto-start through a packaged-app startup task stored per-user (no admin). This sets that task to DisabledByUser so it stays off, and also catches any Copilot Run keys or scheduled tasks a given build uses.

Disable Copilot startup

Takes effect at the next sign-in. Run -List first if you want to preview matches.

powershell -ExecutionPolicy Bypass -File C:\Disable-Copilot-Startup.ps1

Options: -ListPreview only — lists what would change, changes nothing. -EnableUndo — turns Copilot startup back on.

Do it manually instead

Run box: Win+R → ms-settings:startupapps opens Startup apps directly (or taskmgr for Task Manager → Startup apps).

  1. Settings → Apps → Startup.
  2. Find Microsoft 365 Copilot (and/or Copilot) and switch it Off.

Or: Ctrl+Shift+Esc → Task Manager → Startup apps → select the Copilot entry → Disable.

To also hide the taskbar Copilot button (separate from startup): Settings → Personalization → Taskbar → turn off Copilot.

Startup

Stop Malwarebytes at startup

Disable-Malwarebytes-Startup.ps1
Admin for service option Download .ps1

Malwarebytes changed how it starts. Older versions add a Run key, task, or Startup shortcut — those are found and disabled automatically. Current v4/v5 launches its tray (mbamtray.exe) from the MBAMService service via the in-app toggle, with no clean registry switch — so the script gives explicit levers for that case. The tray-off-but-keep-protection option is the in-app toggle (manual, below).

Disable any Windows startup entries it finds

Clears old Run keys, scheduled tasks, and Startup-folder shortcuts for Malwarebytes. Safe default; run -List first to see what exists.

powershell -ExecutionPolicy Bypass -File C:\Disable-Malwarebytes-Startup.ps1

Options: -ListPreview only — lists what would change, changes nothing. -StopTrayNowAlso closes the Malwarebytes tray for this session right now.

Do it manually instead (tray off, protection ON — matches the toggle)
  1. Open Malwarebytes.
  2. Settings (gear) → General (or Security, depending on version).
  3. Under Windows startup, turn Off “Launch Malwarebytes in the background when Windows starts.”

This is the toggle that is On by default after every install — the surgical option that stops the background launch while keeping real-time protection.

Stop Malwarebytes entirely at boot

Sets MBAMService to Manual — this also stops real-time protection. Requires admin. Use only if you want Malwarebytes fully dormant; reverse with -Enable.

powershell -ExecutionPolicy Bypass -File C:\Disable-Malwarebytes-Startup.ps1 -DisableService

Reverse: -DisableService -EnableRestores the service to automatic start — protection comes back at boot.

Do it manually instead
  1. Press Win+R, type services.msc, Enter.
  2. Find Malwarebytes Service (MBAMService), double-click.
  3. Set Startup type to Manual (or Disabled), then Stop the service. Apply.

Self-protection: if it is enabled, Malwarebytes may revert external changes — turn self-protection off in the app first (Settings → Security), then apply.

Maintenance

Open Windows Update, pause, then check

Open-WindowsUpdate.ps1
Admin to pause Download .ps1

Opens Settings > Windows Update, pauses updates for a set number of days, waits, then triggers a check for updates — the routine in one shot. Pausing writes the machine-wide pause window in the registry, so the script self-elevates for that step (the Update screen still opens in your normal session). On machines managed by Group Policy / Intune / WSUS, policy may override the pause.

Open, pause, wait, then check

Default is a 7-day pause and a 10-second wait before checking. Adjust with -PauseDays and -WaitSeconds.

powershell -ExecutionPolicy Bypass -File C:\Open-WindowsUpdate.ps1

Options: -PauseDays 14Pause for a custom number of days (default is 7). -ResumeUn-pause — clears the pause and resumes updates. -NoPauseOpen + check only, no pause (no admin needed).

Do it manually instead

Run box: Win+R → ms-settings:windowsupdate opens Windows Update directly.

  1. Press Win+I → Windows Update.
  2. Next to Pause updates, pick a duration (e.g. Pause for 1 week).
  3. When you are ready, click Check for updates. To resume early, click Resume updates.
Maintenance

Set Windows Update options + active hours

Set-WindowsUpdateOptions.ps1
Admin needed Download .ps1

Turns on the two Advanced-options toggles — Receive updates for other Microsoft products (registers the Microsoft Update service) and Notify me when a restart is required to finish updating — and sets Active hours (default 7 AM–11 PM). Machine-wide settings, so it self-elevates. GPO / Intune / WSUS policy may override these.

Turn both on and set active hours

Defaults to active hours 7 AM–11 PM. Pass -ActiveHoursStart / -ActiveHoursEnd (0–23) for other hours.

powershell -ExecutionPolicy Bypass -File C:\Set-WindowsUpdateOptions.ps1

Options: -ActiveHoursStart 8Active hours start hour, 0–23 (default 7 = 7 AM). -ActiveHoursEnd 22Active hours end hour, 0–23 (default 23 = 11 PM). -AutoActiveHoursLet Windows set active hours automatically by activity, instead of fixed hours. -NoMicrosoftUpdateSkip the “Receive updates for other Microsoft products” toggle. -NoRestartNotifySkip the “Notify me when a restart is required” toggle.

Do it manually instead

Run box: Win+R → ms-settings:windowsupdate-options opens Advanced options directly.

  1. Settings → Windows Update → Advanced options.
  2. Turn On Receive updates for other Microsoft productsSwitches Windows Update into “Microsoft Update,” so it also patches other Microsoft software (Office, Visual Studio, SQL Server) — not just Windows. Example: an Office security fix arrives through Windows Update..
  3. Turn On Notify me when a restart is required to finish updatingWindows pops a notification when an installed update needs a reboot to finish, so it isn’t a surprise. Example: a “Restart to finish installing updates” toast appears instead of a silent scheduled restart..
  4. Under Active hoursThe window when Windows treats the PC as “in use” and won’t auto-restart for updates. Manually = you pick it; Automatically = Windows learns it. Example: 7 AM–11 PM means restarts only happen overnight (11 PM–7 AM)., choose Manually and set the start/end times (or Automatically to let Windows decide).
Maintenance

Launch Disk Cleanup

Open-DiskCleanup.ps1
No admin needed Download .ps1

Opens the built-in Disk Cleanup tool (cleanmgr.exe) for the C: drive so you can pick what to remove. Optional switches open the elevated “system files” view (more categories, including Windows Update cleanup) or run an unattended cleanup of common safe categories with no UI.

Open Disk Cleanup

Opens the normal Disk Cleanup window for C: (use -Drive D for another drive).

powershell -ExecutionPolicy Bypass -File C:\Open-DiskCleanup.ps1

Options: -SystemFilesElevated “system files” view — more categories (Windows Update cleanup, old installs). -AutoUnattended cleanup of common safe categories — no clicks.

Do it manually instead

Run box: Win+R → cleanmgr (or cleanmgr /d C: for a specific drive) opens Disk Cleanup directly.

  1. Press Start, type Disk Cleanup, press Enter.
  2. Choose the C: drive, tick the categories to remove, click OK.
  3. For more (Windows Update cleanup, old installations), click Clean up system files and approve the admin prompt.
