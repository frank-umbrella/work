PC Setup Scripts

PowerShell scripts for the things that have to be re-fixed every time you sign into a new or freshly imaged machine. Each script picks the right mechanism at run time, so the same file works on both Windows versions.

Windows 10 & Windows 11

How to run any of these

  • Save the .ps1 somewhere local, open PowerShell, then run: powershell -ExecutionPolicy Bypass -File .\NAME.ps1
  • -List shows what would change without touching anything · -WhatIf is a full dry run · -Enable reverses a disable.
  • Copied from a network share and blocked? Run Unblock-File .\NAME.ps1 once.
Taskbar

Show all system tray icons

Show-AllTrayIcons.ps1
No admin needed Download .ps1

Force every notification-area icon to show directly on the taskbar instead of hiding in the Windows 11 chevron flyout. Windows 11 made this per-app (IsPromoted under NotifyIconSettings) and new apps keep landing hidden; on Windows 10 it flips the old “always show all icons” switch. Once nothing is hidden the chevron disappears.

Show all icons now

Promotes every current tray icon and restarts Explorer so they appear immediately.

# one-shot
powershell -ExecutionPolicy Bypass -File .\Show-AllTrayIcons.ps1
Do it manually instead
Windows 11
  1. Right-click the taskbar → Taskbar settings.
  2. Expand Other system tray icons.
  3. Turn On the toggle for each app you want always visible. The chevron disappears once nothing is hidden.

Faster: click the chevron to open the flyout, then drag an icon out onto the taskbar — that promotes it instantly.

Windows 10
  1. Right-click the taskbar → Taskbar settings.
  2. Under Notification area, click Select which icons appear on the taskbar.
  3. Turn on Always show all icons in the notification area.

Re-apply automatically on every logon

Copies the script to %LOCALAPPDATA% and registers a logon task that re-runs ~1 minute after each sign-in — so newly installed apps get promoted too, without you remembering to.

powershell -ExecutionPolicy Bypass -File .\Show-AllTrayIcons.ps1 -Install
Do it manually instead
  1. Open Task Scheduler → Create Task.
  2. Triggers tab: New → At log on, delay 1 minute.
  3. Actions tab: New → Program powershell.exe, arguments -NoProfile -ExecutionPolicy Bypass -WindowStyle Hidden -File "C:\path\Show-AllTrayIcons.ps1".
  4. Save. (There is no built-in Settings toggle that survives new app installs — this is the manual equivalent of -Install.)

Remove the logon task

Unregisters the task and deletes the local copy.

powershell -ExecutionPolicy Bypass -File .\Show-AllTrayIcons.ps1 -Uninstall
Do it manually instead
  1. Open Task Scheduler, find ShowAllTrayIcons in the library, right-click → Delete.
  2. Delete the folder %LOCALAPPDATA%\ShowAllTrayIcons.
Startup

Stop Microsoft 365 Copilot at startup

Disable-Copilot-Startup.ps1
No admin needed Download .ps1

“Microsoft 365 Copilot” is the rebranded Microsoft 365 / Office app; “Copilot” is the standalone assistant. Both auto-start through a packaged-app startup task stored per-user (no admin). This sets that task to DisabledByUser so it stays off, and also catches any Copilot Run keys or scheduled tasks a given build uses.

Disable Copilot startup

Takes effect at the next sign-in. Run -List first if you want to preview matches.

powershell -ExecutionPolicy Bypass -File .\Disable-Copilot-Startup.ps1
# preview only:  -List      # put it back:  -Enable
Do it manually instead
  1. Settings → Apps → Startup.
  2. Find Microsoft 365 Copilot (and/or Copilot) and switch it Off.

Or: Ctrl+Shift+Esc → Task Manager → Startup apps → select the Copilot entry → Disable.

To also hide the taskbar Copilot button (separate from startup): Settings → Personalization → Taskbar → turn off Copilot.

Startup

Stop Malwarebytes at startup

Disable-Malwarebytes-Startup.ps1
Admin for service option Download .ps1

Malwarebytes changed how it starts. Older versions add a Run key, task, or Startup shortcut — those are found and disabled automatically. Current v4/v5 launches its tray (mbamtray.exe) from the MBAMService service via the in-app toggle, with no clean registry switch — so the script gives explicit levers for that case. The tray-off-but-keep-protection option is the in-app toggle (manual, below).

Disable any Windows startup entries it finds

Clears old Run keys, scheduled tasks, and Startup-folder shortcuts for Malwarebytes. Safe default; run -List first to see what exists.

powershell -ExecutionPolicy Bypass -File .\Disable-Malwarebytes-Startup.ps1
# preview:  -List      # also kill the tray now:  -StopTrayNow
Do it manually instead (tray off, protection ON — matches the toggle)
  1. Open Malwarebytes.
  2. Settings (gear) → General (or Security, depending on version).
  3. Under Windows startup, turn Off “Launch Malwarebytes in the background when Windows starts.”

This is the toggle that is On by default after every install — the surgical option that stops the background launch while keeping real-time protection.

Stop Malwarebytes entirely at boot

Sets MBAMService to Manual — this also stops real-time protection. Requires admin. Use only if you want Malwarebytes fully dormant; reverse with -Enable.

powershell -ExecutionPolicy Bypass -File .\Disable-Malwarebytes-Startup.ps1 -DisableService
# restore protection:  -DisableService -Enable
Do it manually instead
  1. Press Win+R, type services.msc, Enter.
  2. Find Malwarebytes Service (MBAMService), double-click.
  3. Set Startup type to Manual (or Disabled), then Stop the service. Apply.

Self-protection: if it is enabled, Malwarebytes may revert external changes — turn self-protection off in the app first (Settings → Security), then apply.
