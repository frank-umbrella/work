Email — Resend (ALERT_TO recipient)

Uses the same branded shell as the existing backup-failed / OMSA-warning / disk-low emails, just in the new warn amber palette so the eye reads it as "advisory, not panic." Fires once per host when a target first crosses the threshold — tracked with a per-pcId backupDiskAgeFirstWarnAt field on the PC doc so we don't re-send every check-in.

Firing rules. Once per host per threshold-crossing. The worker writes backupDiskAgeFirstWarnAt on the PC doc when an aged target is first detected; subsequent check-ins skip the email/webhook as long as the same target is still primary and still aged. If the operator swaps the disk (or bumps the threshold past the current age), the field clears and we'll re-arm. Snoozes obey the same alertsSnoozeUntil mute as every other alert — muted hosts won't email/webhook but still tint the dashboard.

What changes in the worker. A new sendBackupDiskAgedEmail() in the same shape as sendOmsaWarningEmail / sendBackupFailureEmail; a new warn entry in SEVERITY_PALETTE; a new event type in eventMeta() so the webhook adapters know what subject + color to use; and the trigger guard inside handleCheckin that compares the freshly-computed primary target's (target, ageDays) against the stored backupDiskAgeFirstWarnAt.
