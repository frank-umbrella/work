Watchtower notification design previews

Three different polish levels for the alert emails and webhook payloads. Pick the one you like and I'll wire it into the worker. Each variant shows all six email types stacked, then a mock of the four chat-app webhook formats below.

A — Minimal Polish

Closest to what's there today. Thin teal accent strip at the top, branded wordmark instead of a bare H2, structured key/value tables with proper labels, severity color on the headline only, muted dashboard CTA. Conservative — looks at home next to GitHub / Stripe / Linear notifications. Lowest risk in Outlook because the layout is still all tables.

B — Hero Card

Bigger visual investment. Full-bleed teal header with the tower mark, then a colored severity band, headline, and a structured fact grid (not tables). One prominent brand-teal CTA button per email. Branded footer bar. Feels more like a SaaS dashboard email (Vercel / Notion / PagerDuty alert style).

The most brand-forward option. Dark-navy hero with a glowing cyan beacon and the full Watchtower mark. Severity gets a pill-shape badge sitting on a light card body. Two-button footer (primary "Open" + secondary "Silence"). Looks like a Datadog / Linear / Vercel critical-alert page emailed to you. Highest visual weight — feels serious.
