Watchtower alert-state icon previews

Three severity states for the Watchtower mark. The healthy tower is the existing icon. The warning amber tower is half-destructed (2 crenellations gone, chunk missing from the top-right wall, vertical crack). The critical red tower is mostly destroyed (only one chipped crenellation left, jagged wall, dark beacon, broken door). All three render at the same 64x64 viewBox so they're drop-in swappable wherever an alert icon is needed.

How the three would be used. Today the dashboard favicon, browser tab, EXE icon, system tray, installer wizard, brand.html, and email/Chat headers all use the OK tower. If we add the alert variants we'd swap the icon dynamically in two surfaces:
  • Browser tab favicon — flips to amber or red if the fleet has any non-OK host. The favicon refresh draws attention back to the tab when something needs you.
  • System tray icon — swap on the next check-in if this host (or in the dashboard's case, the worst fleet member) is in a non-OK state. Operator gets visual feedback without expanding the tray.
Emails/webhooks already convey severity through their headline color + severity band, so the icon there can stay the unchanging brand mark. Note: nothing is being changed yet. This page is just for picking a direction.
