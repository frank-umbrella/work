Appearance

Choose a color theme and text size. Saved to your user profile so the dashboard looks the same on every device you sign in from.

Master webhook URL

All per-agent webhook events (IP changes, backup failures) POST a JSON payload to this URL when their per-PC webhookEnabled toggle is on. Set once for every endpoint.

Per-host drawer lets you set a different URL just for that PC; the master URL above is the default for every endpoint. Reset all hosts to master wipes every per-host override in one shot — useful after consolidating from a multi-URL setup.

Snooze endpoint alerts

Stops the browser-tab favicon from blinking on critical endpoint state until the snooze expires. Doesn't silence emails or webhooks (use the per-host Config toggles for that). Synced across devices for all admins.

Backup disk age alert

Flag hosts whose primary backup disk (the destination most recently written to) has its oldest backup further back than this threshold — i.e. the disk has been the active rotation for that long without being swapped out. Catches drives at end-of-life before they catastrophically fail. Default 2.5 years · consumer USB drives typically degrade past this window. Synced across all endpoints.

Latest agent version (optional override)

After build.ps1 -Publish uploads to GitHub Releases, the worker automatically picks up the latest watchtower-v* tag — Download Installer + Update-available badges + auto-update all "just work" with no paste-in step. Set the fields below only when you want to pin or stage a specific version (e.g. hold back a hotfix while testing). Empty fields = use whatever GitHub Releases has as latest.

Pinned version is older than what GitHub has. Pinning is meant for rollback or staging — if you just want to ship the latest build, click Clear pin.
Agents verify the SHA256 of the downloaded EXE against this value before running it. A mismatch refuses the install — protects against tampered uploads.

Skip versions (auto-update blocklist)

Comma-separated list of agent versions that should be skipped by the auto-updater. The worker's /latest-version filters these out before reporting "latest" so endpoints jump past known-bad releases to the next safe version. Example: 0.14.30, 0.14.31
