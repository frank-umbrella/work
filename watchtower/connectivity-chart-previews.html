Watchtower connectivity chart previews

Four ways to visualize a host's powered + internet uptime in the dashboard drawer. Same sample data across all four — a host that's mostly online, with one 6-hour outage 15 days ago, three smaller blips, and a 2-day powered-off window (vacation). Pick whichever reads best for spotting "did this host blip last week?"

Option A — 30-day daily strip

One cell per day, color tells you whether the day was clean or had problems.

A · Best for at-a-glance "any bad days lately." Tiny vertical footprint. Reads like a Datadog uptime bar. Doesn't show time-of-day — a 30-min outage at 9 AM looks the same as a 30-min outage at 3 PM. Sub-day blips collapse into "amber" without exact timing.
  • Drawer space: ~80px tall total
  • Resolution: per-day
  • Hover tooltip would show exact times

Option B — 7-day hourly heatmap (two lanes)

Top lane is powered state, bottom is internet state. 168 cells across (24h × 7 days). You can see "host was on but offline at 9 AM Tuesday."

B · Best for spotting a specific blip and its exact time. The two-lane stacking is the cleanest way to answer "was the host even on when it went offline?" — if powered shows on but online shows red, you have a real outage. If powered is grey too, the user just shut down the laptop.
  • Drawer space: ~110px tall total
  • Resolution: per-hour
  • Needs 15-min sampling on the agent (Phase 2 work)

C · Best for spotting patterns over weeks — "Friday afternoons are always bad" or "this host hasn't blipped in a month." Familiar from GitHub contribution graphs. Wastes horizontal space (only ~360px wide). Day numbers fit inside but tiny.
  • Drawer space: ~200px tall, ~360px wide
  • Resolution: per-day, with numeric in-cell label
  • Mobile-friendly

Option D — 30-day dual-strip overlay

Two stacked continuous bars across 30 days. Top = powered, bottom = online. Notches/colors mark outages. Compares the two metrics side-by-side at full width.

D · Best for answering "is this a network problem or a power problem?" Side-by-side comparison makes the cause obvious — gaps in BOTH lanes = user turned the machine off; gaps only in the bottom lane = network blip. Sub-day resolution via narrow notches. Most "ops dashboard" feel.
  • Drawer space: ~90px tall total
  • Resolution: positional within 30 days (~6 min per pixel at 640px wide)
  • Hover tooltip would show exact times
