Status icon previews

Six options for the consolidated health pill in the Endpoints table Status column. Each option shows all four states (Healthy / Warning / Critical / Muted) in both light and dark backgrounds, and each pill is rendered with the exact same CSS that ships in production -- this is what you'll see in the dashboard. Pick by number and reply with the choice; I'll wire it in.

What we ship today (v0.14.153): Option 1 -- raw Unicode glyphs. They work but render slightly differently per OS / browser / font (the &#9883; trident on Mac vs Windows is the worst offender). Options 3 through 11 are all non-Unicode -- inline SVG paths or pure CSS shapes that render pixel-identically on every browser, OS, and font setting.
Non-Unicode = no glyph dependency. Each icon below is either (a) inline SVG embedded in the HTML so the browser draws the path itself with no font involvement, or (b) CSS-only shapes built from borders / pseudo-elements / clip-path. Neither path can be affected by OS font fallback, system emoji style, font-substitution from corporate group policy, or a missing font on a fresh Windows Server install. Options 1 and 2 use Unicode codepoints and are subject to all of those variances; everything from option 3 onward is non-Unicode by construction.

My recommendation

Option 3 (Heroicons solid) or option 7 (Material Symbols rounded). Both are well-tested at small sizes, render pixel-identically on every browser / OS / corporate Windows policy, and pair nicely with our typography. Heroicons reads as "modern SaaS dashboard" -- Material Symbols reads as "Google-adjacent, slightly friendlier." Bootstrap (option 10) is a strong third if you want an icon family more commonly found in MSP-vertical tools like ConnectWise / Datto. The pure-CSS option (11) is the most portable but the visual language is narrow -- a check, a triangle, an X, a dash, and that's about it; if we ever add a fifth state we'd have to invent a new shape.
